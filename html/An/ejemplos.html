    🔍 Ejemplos


    A continuación se verá un ejemplo de la estructura de archivos que maneja Angular así como la estructura de sus componentes.


    📁 Estructura Básica

    Antes de comenzar, asegúrate de tener instalados: Node.js y npm: Son fundamentales para gestionar paquetes en JavaScript y ejecutar aplicaciones Node.js. Puedes descargarlos desde nodejs.org.

  • Estructura de un proyecto Angular:

    • Un proyecto Angular típicamente tiene la siguiente estructura:

    mi-primera-app/ ├── e2e/ ├── node_modules/ ├── src/ │ ├── app/ │ │ ├── app.component.css │ │ ├── app.component.html │ │ ├── app.component.spec.ts │ │ ├── app.component.ts │ │ └── app.module.ts │ ├── assets/ │ ├── environments/ │ ├── index.html │ └── main.ts ├── angular.json ├── package.json └── tsconfig.json
    En donde:
  • src: Contiene el código fuente de tu aplicación.
  • app: Contiene los componentes, servicios y módulos principales.
  • angular.json: Configura el proyecto de Angular.
  • package.json: Lista las dependencias del proyecto.
  • tsconfig.json: Configura el compilador de TypeScript.

    • 🛠️ Componente Básico

    A continuación se verá un ejemplo de un componente básico:

    // app.component.ts import { Component } from '@angular/core'; @Component({ selector: 'app-root', templateUrl: './app.component.html', styleUrls: ['./app.component.css'] }) export class AppComponent { title = 'mi-primera-app'; }
    En donde:
  • @Component: Decorador que define un componente.
  • selector: El nombre del selector CSS que se usará para insertar el componente en el HTML.
  • templateUrl: La ruta a la plantilla HTML del componente.
  • styleUrls: La ruta a los archivos CSS del componente.
  • title: Una propiedad del componente que se muestra en la plantilla.
